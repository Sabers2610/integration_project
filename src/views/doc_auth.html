API Autenticación

Endpoints

  • POST /1.0/login/

    Iniciar sesion dentro del sistema

    curl -X POST https://localhost:3001/1.0/login

    • Request Body:

    {
	"email": "usuario@gmail.com",
	"password": "Usuario123**"
}

    Request Body Parameters:

    • email: Type: String. Requerido: Si. Descripción: Email del usuario.
    • password: Type: String. Requerido: Si. Descripción: Password del usuario.

    Respuesta:

    {
        "JWT TOKEN"
    }

  • POST /1.0/register/

    Registrar usuario dentro del sistema

    curl -X POST https://localhost:3001/1.0/register

    • Request Body:

    {
"rut": "11111111-1",
"name": "Raul",
"lastname": "Fuentes",
"age": 20,
"address": "Calle #1, avenida #1",
"email": "RaulFuentes@gmail.com",
"password": "Raul123**",
}

    Request Body Parameters:

    • rut: Type: String. Requerido: Si. Descripción: Rut del usuario.
    • name: Type: String. Requerido: Si. Descripción: Nombre del usuario.
    • lastname: Type: String. Requerido: Si. Descripción: Apellido del usuario.
    • age: Type: int. Requerido: Si. Descripción: Edad del usuario.
    • address: Type: String. Requerido: Si. Descripción: Domicilio del usuario.
    • email: Type: String. Requerido: Si. Descripción: Email del usuario.
    • password: Type: String. Requerido: Si. Descripción: Contraseña del usuario.

    Respuesta:

    {
"rut": "11111111-1",
"name": "Raul Fuentes",
"age": 20,
"direction": "Calle #1, avenida #1",
"email": "RaulFuentes@gmail.com",
"especiality": "N/A",
"employee": "Cliente"
}
