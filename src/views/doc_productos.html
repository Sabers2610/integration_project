API Producto

Endpoints

  • POST /1.0/productos/:id

    Buscar un producto especifico en una sucursal especifica

    curl -X POST -H "Authorization: Bearer JWT_TOKEN -d "id=1" https://localhost:3002/1.0/productos

    • Parámetros:

    • id: Número identificador del producto.

    Autenticación:

    • Type: Bearer Token (JWT)
    • Header: Authorization: Bearer JWT_TOKEN

    Request Body:

    {
	"id_sucursal": 1
}

    Request Body Parameters:

    • id_sucursal: Type: int. Requerido: Si. Descripción:Número identificador de la sucursal.

    Respuesta:

    {
	"producto": {
			"id_producto": "FER-1",
			"nombre_producto": "Tornilladora stanley-15800",
			"marca": "Stanley",

			{
				"stock": 22,

	},
	"historial_precio": [
		{

}

  • POST /1.0/preguntas/

    Ingresa una pregunta al sistema

    curl -X POST -H "Authorization: Bearer JWT_TOKEN https://localhost:3002/1.0/preguntas

    • Request Body:

    {
	"id_categoria": 2,
	"pregunta": "Prueba de pregunta whoknows"
}

    Request Body Parameters:

    • id_categoria: Type: int. Requerido: Si. Descripción:Número identificador de la categoria.
    • pregunta: Type: String. Requerido: Si. Descripción:Pregunta a realizar.

    Respuesta:

    {
	"message": "Pregunta realizada con éxito."
}

  • POST /1.0/preguntas/respuesta/

    Ingresa una respuesta a una pregunta especifica

    curl -X POST -H "Authorization: Bearer JWT_TOKEN https://localhost:3002/1.0/preguntas/respuestas/

    • Request Body:

    {
	"id-pregunta": 2,
	"respuesta": "respuesta de pregunta whoknows"
}

    Request Body Parameters:

    • id_pregunta: Type: int. Requerido: Si. Descripción:Número identificador de la pregunta.
    • respuesta: Type: String. Requerido: Si. Descripción:Respuesta a la pregunta.

    Respuesta:

    {
	"message": "Respuesta registrada"
}
